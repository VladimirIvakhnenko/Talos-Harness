# Руководство по ST для matiec

### 1. Общие принципы и синтаксис ST

#### 1.1. Базовый синтаксис
- ST — текстовый язык высокого уровня, синтаксически похожий на Pascal.
- Каждый **оператор** завершается точкой с запятой `;`.
- Регистр символов **не имеет значения** (например, `ABC` и `abc` эквивалентны).
- Пробелы и переносы строк игнорируются, но **критичны для читаемости**.

#### 1.2. Структура программы
Программа на ST для matiec обычно состоит из:
1. **Раздела объявлений** (`VAR`, `VAR_INPUT`, `VAR_OUTPUT`, `VAR_IN_OUT`)
2. **Тела программы** — исполняемые операторы

```pascal
PROGRAM Example
VAR
    Counter : INT := 0;
END_VAR

Counter := Counter + 1;
END_PROGRAM
```

**Важно:** В matiec тело программы начинается сразу после `END_VAR`, ключевое слово `BEGIN` **не требуется** и может вызвать ошибку компиляции.

#### 1.3. Комментарии
- Строковый комментарий: `// Это комментарий`
- Многострочный комментарий: `(* Это многострочный комментарий *)`

### 2. Типы данных и литералы

#### 2.1. Стандартные типы данных

| Тип | Описание | Диапазон |
|-----|----------|----------|
| `BOOL` | Логический | TRUE / FALSE |
| `BYTE`, `WORD`, `DWORD`, `LWORD` | Битовые строки | 8, 16, 32, 64 бит |
| `SINT`, `INT`, `DINT`, `LINT` | Целочисленные со знаком | 8, 16, 32, 64 бит |
| `USINT`, `UINT`, `UDINT`, `ULINT` | Беззнаковые целые | 8, 16, 32, 64 бит |
| `REAL`, `LREAL` | Числа с плавающей точкой | 32, 64 бит |
| `STRING` | Строка | до 255 символов (по умолчанию) |
| `TIME` | Длительность | TIME#1d2h3m4s5ms |

#### 2.2. Литералы
- **Булевы**: `TRUE`, `FALSE`
- **Целочисленные**: десятичные `1234`, двоичные `2#101010`, восьмеричные `8#12`, шестнадцатеричные `16#1AF5`
- **С плавающей точкой**: `1.234`, `1.234E-5`
- **Типизированные**: `BOOL#1`, `INT#123`, `REAL#1.23`
- **Строковые**: `'Hello'`
- **Временные**: `TIME#1d2h3m4s5ms`, `T#3s`

### 3. Стандартные функции и функциональные блоки matiec

matiec поддерживает стандартную библиотеку IEC 61131-3 (2-е издание, 2001 г.). Ниже приведён перечень основных стандартных блоков и функций.

#### 3.1. Стандартные функции

##### 3.1.1. Функции преобразования типов (Type Conversion Functions)
- `BOOL_TO_INT`, `INT_TO_REAL`, `REAL_TO_INT`, `DINT_TO_TIME`, `TIME_TO_REAL` и т.д.
- Пример: `Level := INT_TO_REAL(RawValue);`

##### 3.1.2. Арифметические функции
- `ABS(x)` — абсолютное значение
- `SQRT(x)` — квадратный корень
- `LN(x)`, `LOG(x)`, `EXP(x)` — логарифмические и экспоненциальные функции
- `SIN(x)`, `COS(x)`, `TAN(x)` — тригонометрические функции

##### 3.1.3. Функции выбора (Selection Functions)
- `MAX(a, b)` — максимум из двух значений
- `MIN(a, b)` — минимум из двух значений
- `LIMIT(min, x, max)` — ограничение значения x в диапазоне [min, max]
- `SEL(g, a, b)` — выбор: если g = FALSE, то a, иначе b

##### 3.1.4. Функции сравнения
- `GT(a, b)` — `a > b`
- `GE(a, b)` — `a >= b`
- `LT(a, b)` — `a < b`
- `LE(a, b)` — `a <= b`
- `EQ(a, b)` — `a = b`
- `NE(a, b)` — `a <> b`

##### 3.1.5. Битовые функции
- `SHL(x, n)`, `SHR(x, n)`, `ROL(x, n)`, `ROR(x, n)` — сдвиги
- `AND(x, y)`, `OR(x, y)`, `XOR(x, y)`, `NOT(x)` — побитовые операции

**Примечание:** matiec поддерживает расширяемые функции (`ADD`, `MUL`, `AND`, `OR`, `XOR`) с переменным числом аргументов (до 499).

#### 3.2. Стандартные функциональные блоки (Standard Function Blocks)

##### 3.2.1. Таймеры (Timers)
| Блок | Описание | Входы | Выходы |
|------|----------|-------|--------|
| `TON` | Таймер с задержкой включения | `IN : BOOL`, `PT : TIME` | `Q : BOOL`, `ET : TIME` |
| `TOF` | Таймер с задержкой выключения | `IN : BOOL`, `PT : TIME` | `Q : BOOL`, `ET : TIME` |
| `TP` | Импульсный таймер | `IN : BOOL`, `PT : TIME` | `Q : BOOL`, `ET : TIME` |

##### 3.2.2. Счётчики (Counters)
| Блок | Описание | Входы | Выходы |
|------|----------|-------|--------|
| `CTU` | Счётчик нарастающий | `CU : BOOL`, `R : BOOL`, `PV : INT` | `Q : BOOL`, `CV : INT` |
| `CTD` | Счётчик убывающий | `CD : BOOL`, `LD : BOOL`, `PV : INT` | `Q : BOOL`, `CV : INT` |
| `CTUD` | Счётчик нарастающий/убывающий | `CU, CD, R, LD : BOOL`, `PV : INT` | `QU, QD : BOOL`, `CV : INT` |

##### 3.2.3. Детекторы фронта (Edge Detection)
| Блок | Описание | Входы | Выходы |
|------|----------|-------|--------|
| `R_TRIG` | Детектор переднего фронта | `CLK : BOOL` | `Q : BOOL` |
| `F_TRIG` | Детектор заднего фронта | `CLK : BOOL` | `Q : BOOL` |

##### 3.2.4. Бистабильные триггеры (Bistable)
| Блок | Описание | Входы | Выходы |
|------|----------|-------|--------|
| `SR` | Set-Reset (доминантный Set) | `S1 : BOOL`, `R : BOOL` | `Q1 : BOOL` |
| `RS` | Reset-Set (доминантный Reset) | `S : BOOL`, `R1 : BOOL` | `Q1 : BOOL` |

### 4. Операторы и выражения

#### 4.1. Арифметические операторы
`+`, `-`, `*`, `/`, `**` (возведение в степень), `MOD` (остаток от деления)

#### 4.2. Логические операторы
- **Рекомендуется**: `AND`, `OR`, `XOR`, `NOT`
- **Не рекомендуется**: `&` (нестандартно, может не поддерживаться)

#### 4.3. Операторы сравнения
`=`, `<>`, `>`, `>=`, `<`, `<=`

#### 4.4. Приоритет операторов (от высшего к низшему)
1. `**`
2. `-` (унарный), `NOT`
3. `*`, `/`, `MOD`
4. `+`, `-` (бинарный)
5. `<`, `>`, `<=`, `>=`
6. `=`, `<>`
7. `AND`
8. `XOR`
9. `OR`

**Рекомендация:** использовать скобки для явного задания приоритета.

### 5. Управляющие конструкции

#### 5.1. Условный оператор IF
```pascal
IF condition THEN
    // statements
ELSIF condition2 THEN
    // statements
ELSE
    // statements
END_IF;
```

#### 5.2. Оператор выбора CASE
```pascal
CASE value OF
    1:      // statements
    2, 3:   // statements
    4..10:  // statements
    ELSE    // statements
END_CASE;
```

#### 5.3. Циклы
**FOR**:
```pascal
FOR i := 1 TO 10 BY 1 DO
    // statements
END_FOR;
```

**WHILE**:
```pascal
WHILE condition DO
    // statements
END_WHILE;
```

**REPEAT**:
```pascal
REPEAT
    // statements
UNTIL condition
END_REPEAT;
```

### 6. Функции и функциональные блоки

#### 6.1. Определение функции (FUNCTION)
```pascal
FUNCTION Add : INT
VAR_INPUT
    a : INT;
    b : INT;
END_VAR
    Add := a + b;
END_FUNCTION
```

#### 6.2. Определение функционального блока (FUNCTION_BLOCK)
```pascal
FUNCTION_BLOCK FB_Counter
VAR_INPUT
    Enable : BOOL;
END_VAR
VAR_OUTPUT
    Count : INT;
END_VAR
VAR
    InternalCount : INT := 0;
END_VAR

    IF Enable THEN
        InternalCount := InternalCount + 1;
    END_IF;
    Count := InternalCount;
END_FUNCTION_BLOCK
```

#### 6.3. Вызов функций и ФБ
```pascal
result := Add(a := 5, b := 10);   // именованные параметры
result := Add(5, 10);             // позиционные параметры
```

### 7. Массивы и структуры

#### 7.1. Массивы
```pascal
VAR
    arr1 : ARRAY[1..10] OF INT;
    arr2 : ARRAY[-5..5, 1..3] OF REAL;
END_VAR

arr1[3] := 100;
arr2[0, 2] := 3.14;
```

#### 7.2. Структуры (STRUCT)
```pascal
TYPE MotorData :
STRUCT
    Speed : REAL;
    Temp  : REAL;
    Status : BOOL;
END_STRUCT;
END_TYPE

VAR
    Motor : MotorData;
END_VAR

Motor.Speed := 1500.0;
```

### 8. Специфические требования для совместимости с matiec

1. **Полная стандартизация**: Используйте только синтаксис IEC 61131-3 2nd Ed. (2001).
2. **Строгие типы**: Избегайте неявных преобразований типов.
3. **Завершающие точки с запятой**: Обязательны для всех операторов.
4. **Регистр**: Регистр не важен, но рекомендуется единый стиль (ключевые слова UPPERCASE, переменные CamelCase).
5. **Отсутствие `BEGIN`**: В matiec тело программы начинается сразу после `END_VAR`.
6. **Пробелы вокруг операторов**: Обязательны для корректного парсинга (`a := b + c;`, не `a:=b+c;`).
7. **Избегайте**:
   - Прямых ссылок на адреса ввода-вывода (`AT %IX...`)
   - Нестандартных операторов (`&` вместо `AND`)
   - Синтаксиса 3-го издания стандарта

### 9. Практические правила для надёжной совместимости с matiec

Ниже приведены универсальные правила, которые повышают устойчивость ST-кода к особенностям парсинга `matiec` и одновременно делают код более предсказуемым для генерации.

#### 9.1. Используйте безопасные имена переменных

Старайтесь использовать короткие, технические и однозначные идентификаторы. Чем проще имя, тем ниже риск конфликтов с диалектными особенностями, служебными словами или неоднозначным разбором.

- **Предпочитайте** краткие имена:
  - `Err`, `PrevErr`
  - `SeqStep`, `State`
  - `In1`, `In2`, `Out1`
  - `Temp1`, `LimitHi`, `LimitLo`
- **Избегайте**:
  - слишком длинных описательных идентификаторов без необходимости
  - имён, близких к ключевым словам языка
  - разных переменных с очень похожими именами

Пример безопасного объявления:

```pascal
VAR
    Offset : REAL := 2.0;
    SeqStep : INT := 0;
    Err : REAL := 0.0;
    PrevErr : REAL := 0.0;
END_VAR
```

#### 9.2. Не оставляйте арифметику "впритык" к сравнению

Выражения вида `A < B - C` и `A > B + C` лучше не оставлять в компактной форме. Более надёжный стиль: скобки или промежуточные переменные.

- **Предпочтительно**: `Value1 < (Limit1 - Offset);`
- **Предпочтительно**: `Value1 > (Limit1 + Offset);`
- **Ещё лучше**: заранее вычислить пороги в отдельных переменных

```pascal
VAR
    LimitOff : REAL;
    LimitOn : REAL;
END_VAR

LimitOff := Limit1 - Offset;
LimitOn := Limit1 + Offset;

IF Value1 >= Limit1 THEN
    Flag1 := TRUE;
ELSIF Value1 < LimitOff THEN
    Flag1 := FALSE;
END_IF;
```

#### 9.3. Для вычислительных блоков используйте короткие внутренние имена

Для внутренних вычислений удобны краткие и повторяемые шаблоны имён:

- `Err`
- `PrevErr`
- `Calc1`
- `RawOut`
- `Ts`

Это снижает риск конфликтов и делает код стабильнее для генерации.

```pascal
VAR
    Err : REAL;
    Accum : REAL := 0.0;
    PrevErr : REAL := 0.0;
    Calc1 : REAL;
    Ts : REAL;
    RawOut : REAL;
END_VAR
```

#### 9.4. Для пошаговой логики лучше `State` или `SeqStep`

Для `CASE`-автоматов и пошаговой логики полезно придерживаться предсказуемой схемы именования.

- **Рекомендуется**: `State`, `SeqStep`, `CurrentState`
- Для сложной логики лучше `ENUM` + `CASE`
- Для простой последовательности допустим `INT` + `CASE`

```pascal
TYPE StateType : (ST_IDLE, ST_RUN, ST_DONE, ST_FAULT);
END_TYPE

VAR
    State : StateType := ST_IDLE;
END_VAR
```

#### 9.5. Если компилятор "сыпется" серией ошибок, сначала проверьте имя первой переменной

Если после одного объявления появляется много ошибок подряд, причина часто не в десятке следующих строк, а в **первом конфликтующем идентификаторе**. Порядок проверки:

1. Имя переменной
2. Тип переменной
3. Инициализация `:=`
4. Арифметика в ближайшем `IF` / `ELSIF`

### 10. Рекомендации по форматированию для RAG-генерации

- **Пробелы вокруг бинарных операторов**: `a := b + c;`
- **Пробел после запятой**: `Add(a, b);`
- **Нет пробела** перед `;`, после `(` и перед `)`
- **Отступы**: 4 пробела на уровень вложенности
- **Длина строки**: максимум 80 символов
- **Использование TRUE/FALSE**: всегда `TRUE` и `FALSE`, не `1` и `0`
- **Безопасные идентификаторы**: предпочитать короткие технические имена вроде `Err`, `State`, `SeqStep`, `RawOut`
- **Промежуточные переменные для порогов**: лучше `LimitOff := A - B;`, чем арифметика прямо в `ELSIF`
